01 The Pair Hunter LC #1 Easy ▶

Given an integer slice nums and a target, return the indices of the two numbers that add up to target. Exactly one valid answer is guaranteed — find it in a single pass.

  • You may not use the same element twice.
  • For each number, ask: "have I seen the value I need to complete the pair?" A hash map answers that in O(1).
  • Store value → index as you walk forward; check before inserting.
  • Duplicates like [3, 3] are valid — your map must not overwrite an index before checking against it.
Expected Function
func twoSum(nums []int, target int) []int {

}
Usage
func main() {
    fmt.Println(twoSum([]int{2, 7, 11, 15}, 9))  // [0 1]
    fmt.Println(twoSum([]int{3, 2, 4},       6))  // [1 2]
    fmt.Println(twoSum([]int{3, 3},          6))  // [0 1]
}
$ go run .
[0 1]
[1 2]
[0 1]
Constraints
2 ≤ nums.length ≤ 104
Exactly one valid solution exists
You may not use the same element twice
Time: O(n) Space: O(n)
Allowed & Forbidden
Allowed
map[int]int single for loop map lookup ok idiom
// seen[complement] → check first, then store index
Forbidden
sort package nested loops O(n²) brute force
// O(n²) nested loop is explicitly not acceptable
💡 Key Insight
At each index i, the complement you need is target - nums[i]. Check if that complement is already in your map. If it is, you're done. If not, store nums[i] → i and move on. One pass, one map.
02 The Duplicate Detector LC #217 Easy ▶

Given an integer slice nums, return true if any value appears at least twice, and false if every element is distinct.

  • An empty slice or single-element slice has no duplicates — return false.
  • You need O(1) average-case lookups — a hash set is the right tool.
  • As you walk the slice, ask "have I seen this before?" before inserting each element.
  • A nested-loop O(n²) approach is not acceptable.
Expected Function
func containsDuplicate(nums []int) bool {

}
Usage
func main() {
    fmt.Println(containsDuplicate([]int{1, 2, 3, 1}))                   // true
    fmt.Println(containsDuplicate([]int{1, 2, 3, 4}))                   // false
    fmt.Println(containsDuplicate([]int{1, 1, 1, 3, 3, 4, 3, 2, 4, 2})) // true
}
$ go run .
true
false
true
Constraints
1 ≤ nums.length ≤ 105
Empty or single-element slice → return false
Time: O(n) Space: O(n)
Allowed & Forbidden
Allowed
map[int]bool map[int]struct{} single for loop
// seen[v] = true — O(1) per element
Forbidden
sort package nested loops
// Sorting is O(n log n) — not acceptable here
💡 Key Insight
Use a map[int]bool as a visited set. For each element, check if it's already in the map — if yes, return true immediately. Otherwise add it and continue. If you finish the loop, return false.
03 The Letter Spy LC #242 Easy ▶

Given two strings s and t, return true if t is an anagram of s. An anagram uses every original letter exactly once, just rearranged.

  • Different lengths → immediately return false.
  • Both strings contain only lowercase English letters.
  • A fixed-size [26]int frequency array counts as O(1) space — use it instead of a map.
  • Count up for s, count down for t; any non-zero slot at the end means not an anagram.
Expected Function
func isAnagram(s string, t string) bool {

}
Usage
func main() {
    fmt.Println(isAnagram("anagram", "nagaram")) // true
    fmt.Println(isAnagram("rat",     "car"))     // false
    fmt.Println(isAnagram("listen",  "silent"))  // true
}
$ go run .
true
false
true
Constraints
1 ≤ s.length, t.length ≤ 5×104
s and t consist of lowercase English letters only
Time: O(n) Space: O(1)
Allowed & Forbidden
Allowed
[26]int array byte arithmetic len() range loop
// freq[c - 'a']++ is the core operation
Forbidden
sort package map[byte]int strings package unicode package
// Fixed-size array → O(1) space, no map needed
💡 Key Insight
In Go, 'a' is a byte with value 97. So c - 'a' maps each letter to an index 0–25 — a perfect fit for a [26]int array. Increment for s, decrement for t, then verify every slot is zero.
04 The Anagram Architect LC #49 Medium ▶

Given a slice of strings strs, group all strings that are anagrams of each other into the same subslice. The output groups and the strings within each group may appear in any order.

  • An empty string forms its own group of one.
  • All strings contain only lowercase English letters.
  • The challenge: design a canonical key two strings share if and only if they are anagrams — without sorting.
  • A fixed-size [26]int frequency array, converted to a string, is the key.
Expected Function
func groupAnagrams(strs []string) [][]string {

}
Usage
func main() {
    fmt.Println(groupAnagrams([]string{"eat", "tea", "tan", "ate", "nat", "bat"}))
    fmt.Println(groupAnagrams([]string{""}))
    fmt.Println(groupAnagrams([]string{"a"}))
}
$ go run .
[[eat tea ate] [tan nat] [bat]]
[[]]
[[a]]
Constraints
1 ≤ strs.length ≤ 104
0 ≤ strs[i].length ≤ 100
strs[i] consists of lowercase English letters only
Empty strs → return [][]string{}
Time: O(n·k) Space: O(n·k)
Allowed & Forbidden
Allowed
map[string][]string [26]int key fmt.Sprintf key string(key[:])
// fmt.Sprintf("%v", freq) turns [26]int into a string key
Forbidden
sort package sort individual strings
// Sorting each string is O(k log k) — use freq array instead
💡 Key Insight
For each string, build a [26]int frequency array, then convert it to a string with fmt.Sprintf("%v", freq). Use that string as a map key. All anagrams produce the same key and land in the same bucket. Collect the map values at the end.
05 The Frequency Podium LC #347 Medium ▶

Given an integer slice nums and an integer k, return the k most frequently occurring elements. The answer may be returned in any order.

  • k is always valid: 1 ≤ k ≤ unique elements.
  • Your solution must be better than O(n log n) — no heap, no sort.
  • The path to O(n) is bucket sort by frequency: index i holds all elements appearing exactly i times.
  • Walk the bucket slice from highest to lowest, collecting until you have k values.
Expected Function
func topKFrequent(nums []int, k int) []int {

}
Usage
func main() {
    fmt.Println(topKFrequent([]int{1, 1, 1, 2, 2, 3}, 2))              // [1 2]
    fmt.Println(topKFrequent([]int{1}, 1))                             // [1]
    fmt.Println(topKFrequent([]int{4, 4, 4, 6, 6, 2, 2, 7}, 3))     // [4 6 2]
}
$ go run .
[1 2]
[1]
[4 6 2]
Constraints
1 ≤ nums.length ≤ 105
1 ≤ k ≤ number of unique elements
The answer is guaranteed to be unique
Time: O(n) Space: O(n)
Allowed & Forbidden
Allowed
map[int]int freq [][]int buckets reverse bucket walk
// buckets[freq] = append(buckets[freq], num)
Forbidden
sort package heap / priority queue O(n log n) strategy
// Bucket sort sidesteps all comparison-based sorting
💡 Key Insight
No element can appear more than n times, so a bucket slice of size n+1 is always enough. Build a frequency map first, then scatter elements into buckets. Walk backwards from index n collecting elements into your result until you have k.
06 The Product Ghost LC #238 Medium ▶

Given an integer slice nums, return a slice output where output[i] is the product of every element in nums except nums[i]. No division allowed.

  • The input always contains at least two elements.
  • The slice may contain zeros — division-based approaches fail silently here.
  • output[i] = (product of all elements left of i) × (product of all elements right of i).
  • Two passes: fill prefix products left-to-right into the output slice, then multiply in suffix products right-to-left with a single running variable.
Expected Function
func productExceptSelf(nums []int) []int {

}
Usage
func main() {
    fmt.Println(productExceptSelf([]int{1, 2, 3, 4}))        // [24 12 8 6]
    fmt.Println(productExceptSelf([]int{-1, 1, 0, -3, 3}))   // [0 0 9 0 0]
    fmt.Println(productExceptSelf([]int{2, 3, 4, 5}))        // [60 40 30 24]
}
$ go run .
[24 12 8 6]
[0 0 9 0 0]
[60 40 30 24]
Constraints
2 ≤ nums.length ≤ 105
The slice may contain zeros
Division is explicitly forbidden
Time: O(n) Space: O(1) extra
Allowed & Forbidden
Allowed
output slice (not counted) two for loops running suffix var
// Pass 1: prefix. Pass 2: suffix × output[i] in place
Forbidden
division operator / math package sort package auxiliary arrays
// No prefix array and suffix array — only one running var
💡 Key Insight
Pass 1 (left→right): output[i] = output[i-1] * nums[i-1] — each slot accumulates the product of everything to its left. Pass 2 (right→left): maintain a suffix variable and multiply output[i] *= suffix; suffix *= nums[i]. No extra array needed.
07 The Chain Walker LC #128 Medium ▶

Given an unsorted integer slice nums, return the length of the longest sequence of consecutive integers. Elements don't need to be adjacent or ordered in the input.

  • Empty slice → return 0.
  • Duplicate values must be handled gracefully — they do not extend a sequence.
  • Load all values into a hash set for O(1) lookups.
  • Only begin counting at a number n if n-1 is not in the set — marking it as a true sequence start.
  • Negative integers are valid input.
Expected Function
func longestConsecutive(nums []int) int {

}
Usage
func main() {
    fmt.Println(longestConsecutive([]int{100, 4, 200, 1, 3, 2}))             // 4  → [1,2,3,4]
    fmt.Println(longestConsecutive([]int{0, 3, 7, 2, 5, 8, 4, 6, 0, 1}))  // 9  → [0..8]
    fmt.Println(longestConsecutive([]int{9, 1, 4, 7, 3, -1, 0, 5, 8, -1, 6})) // 7 → [3..9]
}
$ go run .
4
9
7
Constraints
0 ≤ nums.length ≤ 105
Negative integers are valid
Duplicates are valid — they don't extend sequences
Time: O(n) Space: O(n)
Allowed & Forbidden
Allowed
map[int]bool set two passes forward walk per start
// Each element visited at most twice across entire algorithm
Forbidden
sort package O(n log n) strategy
// Sorting would work logically but violates the O(n) constraint
💡 Key Insight
The "start of sequence" check — !set[n-1] — is what keeps this O(n). If you only start counting when there's no predecessor, each element is walked at most once during a forward scan. Without this guard, you'd recount overlapping chains and blow up to O(n²).
08 The Lost Number LC #268 Easy ▶

Given a slice nums containing n distinct integers in the range [0, n], return the one number that is missing from the sequence. No sorting, no extra space.

  • Empty slice (n = 0) → return 0.
  • All values are distinct and guaranteed in range [0, n].
  • Think about the expected sum of 0 + 1 + … + n vs the actual sum of the slice.
  • An XOR-based approach is equally valid and immune to integer overflow.
Expected Function
func missingNumber(nums []int) int {

}
Usage
func main() {
    fmt.Println(missingNumber([]int{3, 0, 1}))                   // 2
    fmt.Println(missingNumber([]int{0, 1}))                      // 2
    fmt.Println(missingNumber([]int{9, 6, 4, 2, 3, 5, 7, 0, 1})) // 8
}
$ go run .
2
2
8
Constraints
n = nums.length; nums contains n distinct values from [0, n]
Empty slice (n = 0) → return 0
Time: O(n) Space: O(1)
Allowed & Forbidden
Allowed
Gauss formula n*(n+1)/2 XOR reduction single loop
// expected = n*(n+1)/2; return expected - sum(nums)
Forbidden
sort package hash map auxiliary slice
// O(1) space means no extra data structures at all
💡 Key Insight
The sum of integers from 0 to n is n*(n+1)/2. Subtract the actual sum of nums and you get the missing value. For large inputs, the XOR approach result ^= i ^ nums[i] avoids any overflow risk entirely.
09 The Double Agent LC #442 Medium ▶

Given an integer slice nums of length n where every element satisfies 1 ≤ nums[i] ≤ n, return all elements that appear exactly twice. No auxiliary data structures — the input slice is your only workspace.

  • Each value either appears once or twice — never more than twice.
  • The constraint 1 ≤ nums[i] ≤ n is critical: every value is a valid index when decremented by one.
  • Use the sign of each element as a visited flag: when processing value v, negate nums[abs(v)-1]. If it's already negative, abs(v) is a duplicate.
  • You may temporarily mutate nums — the sign-flip approach leaves all absolute values intact.
Expected Function
func findDuplicates(nums []int) []int {

}
Usage
func main() {
    fmt.Println(findDuplicates([]int{4, 3, 2, 7, 8, 2, 3, 1})) // [2 3]
    fmt.Println(findDuplicates([]int{1, 1, 2}))               // [1]
    fmt.Println(findDuplicates([]int{1}))                     // []
}
$ go run .
[2 3]
[1]
[]
Constraints
1 ≤ nums.length ≤ 105
1 ≤ nums[i] ≤ n (length of nums)
Each value appears once or exactly twice
Returned slice may be in any order
Time: O(n) Space: O(1) extra
Allowed & Forbidden
Allowed
sign-flip technique abs() helper inline mutating nums result slice (output)
// abs() is not in stdlib for int — write it inline
Forbidden
sort package hash map frequency array any auxiliary slice
// O(1) extra space — the output slice doesn't count
💡 Key Insight
For each element, compute idx = abs(v) - 1. Negate nums[idx]. If it was already negative before you negated it, that means index idx+1 was visited twice — append abs(v) to results. The sign encodes "visited" without any extra memory.
